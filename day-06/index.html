Accessibility and Formatting Tags

Focus: Chrome DevTools Accessibility panel, and how semantic formatting affects screen readers.

Why semantics matter

Screen readers and the Accessibility tree care about meaning, not just visuals. Some tags describe meaning (strong, del), while others are only visual (b, i, s).

Use DevTools → Elements → Accessibility icon to compare how each example is announced.

Semantic formatting (meaningful)

strong - importance
Please read the updated schedule today.
em - emphasis
This is not the final answer.
del - deleted text
I am a junior senior engineer.
s - strikethrough style. Use del instead.
This is crossed out text.
ins - inserted text
You are now an alumni of the program.
sub - subscript
Water is H2O.
sup - superscript
The formula is x2 + y2.

Presentational formatting (visual only)

b - bold style only
This is bold but not more important.
i - italic style only
This is italic without emphasis.
u - underline style
This is underlined text.
small - smaller font
This is fine print only.
big - larger font (obsolete)
This is larger text.

These tags usually do not add meaning in the Accessibility tree.

Try this in DevTools

  • Inspect each example and open the Accessibility panel.
  • Compare how strong vs b is announced.
  • Compare how del and ins appear in the tree.
